3 key ideas in operating systems:
- virtualization
- concurrency
- persistance

A software responsible for running programs (single or multiple), allows to share resources, enabling programs to interact with peripheral devices.

Virtualization:
Conversion og raw resources into easy to use, generalized and more powerful form

OS is also known as Virtual Machine

OS provides a standard library to applications to perform actions such as sharing of memory ... 

OS is also know as resource manager

Virtual Address Space:
A virtual memory/space allocated to a process to store runtime data

Program:
Set of instructions

Process:
An executing/running instance of a program is a process

User Space:
An environment where user facing applications runs and resides

Kernel:
A part of OS responsible for managing resources and provide system calls

Kernel Space:
An environment where kernel resides

System Call:
A controlled entrypoint that allows a user program to request service from the OS.

What happens from typing './server' until it starts running:
1. Shell reads command
2. Check Permissions
3. Validate presented instructions
4. Allocates Resources
5. Linking and loading of libraries 
6. Execution begins
